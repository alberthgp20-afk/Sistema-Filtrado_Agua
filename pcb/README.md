# PCB

## Justificación

Durante el desarrollo del prototipo se consideró inicialmente la posibilidad de diseñar una tarjeta de circuito impreso (PCB) para integrar los componentes electrónicos del sistema. Sin embargo, debido al tiempo disponible para el desarrollo del proyecto y a la necesidad de realizar constantes pruebas y modificaciones durante la etapa de implementación, se optó por utilizar una protoboard y cableado convencional.

Esta decisión permitió reemplazar componentes, corregir conexiones y realizar ajustes en los sensores y el sistema de bombeo de manera rápida, sin necesidad de fabricar una nueva tarjeta cada vez que se presentaba una modificación.

Además, el objetivo principal del proyecto fue validar el funcionamiento del sistema automatizado de tratamiento y monitoreo del agua, priorizando la correcta integración de los sensores, las bombas de agua, el ESP32 y la pantalla de visualización antes de realizar un diseño electrónico definitivo.

Como trabajo futuro, se propone diseñar una PCB personalizada que permita reducir el tamaño del sistema, mejorar la organización del cableado, aumentar la confiabilidad de las conexiones y facilitar el mantenimiento e instalación del prototipo.
